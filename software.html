Randbay

Project Summary

Randbay is a weekend project I developed in 2012 based upon a silly idea I had with some friends - What if you could buy completely random products off the internet for a given price?


The application takes a set purchase price decided by the user, uses a series of complex SQL queries to locate a random object for sale which is near that price and matches the user request, and then uses the Paypal REST API to purchase it using the linked user Paypal account. The user does not find out what they have bought until the transaction is complete.


The app also incorporates a complex backend which uses cron jobs and the Laravel scheduler to scan the database for items which require moderation, send automated mail, update daily currency conversion rates and populate mailing lists.

Project Summary

Cast is a work in progress DApp which intends to deliver a fully decentralised peer compilation engine, which allows active peer review and collaborative compilation of documents and academic papaers. The application runs on the Rinkeby Ethereum testnet, using ERC20 tokens and smart contracts compiled in Solidity which execute on the Ethereum Virtual Machine (EVM). The DApp is primarily build in Truffle.js and uses web3.js to communicate with the chain. The lightweight HTML and javascript frontend is hosted on the decentralised Ethereum swarm engine (:bzz protocol), meaning that all files for running Cast are encrypted on the blockchain and cannot be tampered with or modified.


The system uses the open source Quill.js templating engine to construct documents from individual building blocks called Deltas. Users may review the documents by marking up areas of the text and suggesting changes. Changes are voted on by other users and either incorporated as new Deltas, or dismissed based upon the consensus of reviewers.


Documents are stored in a smart contract as an array of hash addresses for the relevant Deltas needed to build the document. This means that each document and its entire edit history are all recorded in an encrypted, decentralised instruction template and cannot be controlled or edited by any single user.


Please Note: You will need to install the MetaMask DApp browser addon and a compatible Ethereum wallet to access Cast

Quicko

Project Summary

Quicko is a weekend project I developed in 2014 to demo a peer-to-peer postal platform. The application is written primarily in the php laravel framework with a complex Object Oriented javascript frontend. The app uses the Google Maps REST API and MySQL Spatial Indices to match people who are making journeys to people who have things to deliver to nearby places.


Users record their journeys in advance, allowing people with parcels to arrange for a nearby user to deliver on their behalf. The app features a messaging system which allows users to negotiate prices, pickup locations and other details.

Tiprindi

Project Summary

Tiprindi is a work in progress project which intends to deliver a demonstrator for a blockchain based streaming service. The user interface is built using a conventionally hosted php and javascript stack, with the mp3 objects being located on the Ethereum blockchain via the Swarm engine. Audio is streamed from the chain using the Wowza VOD streaming engine.


Artists receive tips from listeners on a Pay What You Like basis. These tips are processed via a series of smart contracts on the Ethereum blockchain.


Please note: Tiprindi is under active early stage development and so will be periodically down for maintenance.

ML4X

Project Summary

ML4X (Machine Learning Forex) is an ongoing work in progress project built in Python and hosted on the Google Colab TensorFlow cloud. The codebase currently features a simple Flask-ngrok based frontend in order to demonstrate its features. ML4X is an attempt to use neural network machine learning to unify technical and sentiment analysis into a single TensorFlow model. The application uses a recurrent Long-Short Term Memory (LSTM) neural network to predict currency pair movements based upon past technical data. The application also uses web scraping to feed a convolutional neural network and correlate technical data with language based sentiment analysis. This allows sentiment positions from trader publications, blogs, news sites etc. to be mapped on actual technical price movements and patterns to be detected, learned and predicted.


The latest ML4X version is held on a Google Colab notebook. Please follow the instructions in the notebook to run the code and bootstrap the ngrok frontend.


Please note: ML4X is under active early stage development and so will be periodically down for maintenance, updates etc.

DarWin

Project Summary

DarWin is an evolutionary algorithm written in vanilla JavaScript which uses genetic methodology to optimise a structural beam. The project was adapted from a MATLAB script which I wrote when working in aerospace. The algorithm defines a set number of parameters which it seeks to optimise (weight, material, principal stress, cross-sectional size, etc.) and bundles these together into a cost function. Random structures are generated which are assessed against the cost function. This random set comprises a 'generation'.


Each generation only passes on the top 10% of solutions as assessed against the cost function. The next generation combines 'mutations' of these top 10% as well as 'crossovers' which randomly take features from across the gene-pool, and 'wildcards' which inject new genetic data.
